## Learned User Preferences

- Never assume, guess, or speculate — verify, confirm, and validate with codebase evidence, logs, screenshots, root or terminal output before reporting or fixing issues.
- When implementing attached plans: do not edit the plan file; use existing todos (do not recreate); mark items in progress; finish all todos; then re-check the plan for missed items, gaps, duplications, and dead code.
- Before large features or refactors: analyze the full codebase and ask brief, concise clarifying questions first.
- Preserve existing Shell Mono styling, layout, and UX patterns when fixing bugs or adding features — do not redesign unrelated surfaces.
- Do not hardcode provider or model capabilities; dynamically discover models, context windows, and thinking/reasoning support via provider APIs.
- When implementing provider or model features, research current (2026) docs and APIs online rather than relying on stale assumptions.
- Model picker UX: group providers into categories, left-align model IDs, and place thinking-effort controls inline next to each model (not a separate dropdown).
- Take the time needed for thorough work; keep features in structured, modular files and folders rather than monolithic files.
- Create git commits only when the user explicitly asks.
- Remove Cursor-branded or Cursor-style UI references from Vyotiq where the user has corrected this.
- Bug and screenshot analysis should be end-to-end (conversation logs, app logs, code paths) with root causes verified.
- Do not revive legacy phased-execution types/UI (`phase` / `phase-gate` / `phase-ledger-entry`, PhaseLogRow/PhaseLedgerRow) or the removed reflective-autonomy subsystem (terminal gates, goal mode, `plan` tool, harness `05-reflection-autonomy.md`).

## Learned Workspace Facts

- Vyotiq is the product; Agent V is a single solo agent with direct tool access — no sub-agent or delegation architecture. Behavior is governed by natural-language harness markdown in `src/main/harness/`; tools live under `src/main/tools/` with exposure in `src/main/tools/policy/`; orchestrator loop in `src/main/orchestrator/loop/`.
- Stack: Electron (main/renderer IPC), React, TypeScript, Vite 8 (electron-vite 6), Zustand; Tailwind CSS v4 CSS-first tokens in `src/renderer/index.css` (`@theme`, no `tailwind.config.js`). Shell Mono: stealth-dark oklch palette, Geist Sans/Mono, chromeless timeline, frameless workbench (left dock + main canvas). Primary dev OS is Windows (PowerShell). `project.md` is authoritative. Runtime data/logs: `%APPDATA%\vyotiq\vyotiq\`.
- AI providers use raw HTTP only (no vendor SDKs); models via `GET /v1/models`. Adaptive account poller (5s active / 60s idle) fetches host-aware billing and pushes `providers:account-updated`; discovery poller pushes `providers:models-updated` with pricing/context/thinking. `recentBillingBlock.ts` caches billing failures per provider+model for 5 minutes (`BILLING_BLOCK_TTL_MS`) so switching models is never blocked by a sibling model's error.
- Timeline UI: `src/renderer/components/timeline/`; state in `useChatStore` / `useTimelineUiStore`. Hidden tools: `finish` and `ask_user` settle in the log but do not render activity rows (`isTimelineHiddenTool`); `ask_user` uses `ask-user-prompt` rows; answers as compact user bubble. Edit diffs: per-file `FileChangeCard` + `SnippetDiffBody`; unified `DiffViewer` for review line-pick only.
- Prompt/context caching: layered topology in `buildContextLayers.ts` (static system → few-shot → workspace → history → runtime → turn); provider hints in `src/main/providers/cacheHints/`; metrics in `TokenUsage`; Settings → Agent behavior → Prompt caching; audit in `docs/prompt-caching-audit.md`.
- Workbench shell (`WorkbenchShell`): chat left; editor/terminal/preview open a resizable right pane with `[Terminal | Globe | file tabs…]`. Left dock stays visible; `Mod+W` close tab, `Mod+Alt+ArrowUp/Down` cycle. Terminal detach is renderer-only; PTY persists for agent `bash` (`ensureWorkspacePty` / `terminal:attach`).
- Custom keybindings via `settings.ui.keybindings`; `Mod+S` saves dirty editor; `Mod+J` focuses composer (skipped when settings open). Focus: `escapeLayerStack.ts` priority; AskUser soft Tab trap; workbench tab cycle DOM-focuses terminal/editor/browser. Main wakes window on `ask_user`, settlement, scheduled enqueue, second-instance (`requestUserAttention.ts`). Settings close honors user `closeSettings` binding (Escape-only uses escape stack).
- Composer follow-ups: active run Send → steering lane; Queue → queued lane (before finish). During `ask_user`, Send submits answers; Queue defers follow-ups. Stop aborts run without clearing queue; steering persists (tray **prior run** badge). Shortcuts: `Mod+Shift+Enter` Queue, `Escape` Stop. (`src/main/followUps/`, `src/renderer/components/composer/followUps/`). Scheduled runs enqueue into queued lane when conversation is busy (`src/main/scheduler/`).
- Screen capture & vision: composer camera picker (`CaptureScreenButton`, `useCapturePicker`) — user-initiated; narrow ~360px button-anchored panel (not full shell width); lazy JPEG previews; full fidelity via `getUserMedia` + `capture:ingest-frame`; agent `capture` tool uses `capture:request-frame` bridge; optional title redaction in Settings → Screen capture. Multimodal vision: composer attachments (paste/drop/capture) → `prepareVisionParts` / `runVisionQueue`; provider `inputModalities` from discovery; vision tokens in context budget. Drag-drop must not fall back to OS clipboard on failed drop; paste uses clipboard intentionally. Workspace-relative capture paths preserved via `attachments:ingestPaths` + `resolveAttachmentInWorkspace`.
- HTML reports: Settings → Agent behavior → Reports (`settings.ui.reports`) — auto-open, in-app BrowserWindow, host ask_user gate after large edits, token-costing AI footer; free Quick summary via `reports:generate-run-summary`. Auto-open only for live settlements, not transcript replay.
- Editor LSP: optional stdio bridge for in-app CodeMirror (Pyright + TS LS via Electron-as-Node); Settings → Editor LSP; not wired to agent `search`/`read`. Vector re-index: Settings → Vector memory; embedder changes trigger `reindexAllWorkspacesIfVectorMemoryChanged`. Memory append: `memory:write` mode `append` + Settings → Memory UI. `read`/`edit`/editor preserve BOM/EOL via `decodeDiskText.ts`.
- Legacy phased execution removed: engine, harness `05-phased-execution.md`, and timeline rows purged; `normalizeLegacyTranscript.ts` strips `phase` / `phase-gate` / `phase-ledger-entry` on load. Tool re-run and tool permissions removed entirely. Supply chain: pnpm 11 with `minimumReleaseAge` (7 days), `blockExoticSubdeps`, `allowBuilds` in `pnpm-workspace.yaml`; hoisted `node-linker`; frozen `pnpm install --frozen-lockfile` in CI/release (`docs/supply-chain-security.md`); avoid `pnpm dlx` / `npx` for installs.
