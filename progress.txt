## Hotfix (2026-05-28) — production boot React #185 at App

**Root cause:** Phase 3 batched `enabledProviderIds: selectEnabledProviderIds(...)` inside a `useShallow` object pick. Shallow compare uses `Object.is` on each property — the id array is a new reference on every provider-store write (`discoverCached` sets `lastDiscoveredAt`). Boot discover effect re-ran after each discovery → infinite `setState` loop → minified React error #185.

**Fix:** Split provider subscriptions; subscribe to enabled ids with `useShallow((s) => selectEnabledProviderIds(s.providers))` so array elements are compared, not array identity.

**Files:** `src/renderer/App.tsx`, `tests/renderer/app/appBootDiscoverStable.test.ts` (new).

**Tests:** `npm run typecheck` OK; `vitest run tests/renderer/app/appBootDiscoverStable.test.ts` (+ related store/mirror tests) 5/5 pass.
